# Creative Studio 规划文档

> 本文档定义 Ymcp Creative Studio 的技术选型、功能规划和实现路径。

## 1. 项目概述

Creative Studio 是一个面向视觉创作的本地工作台，整合 AI 生图、视频处理、素材编辑等功能，提供可复用的工作流和会话管理能力。

### 1.1 核心目标

- **可复用工作流**：预设参数一键重复执行
- **AI 生图**：文生图、图生图，支持多后端适配
- **视频处理**：纯前端视频转序列帧（参考现有 v2f 实现）
- **素材编辑**：抠图、SpriteSheet 拆分/合成、图片拼接等
- **会话管理**：本地持久化生成历史和素材库

### 1.2 设计原则

- **离线优先**：所有数据本地存储，无需联网
- **可插拔架构**：AI 接口适配器模式，随时切换后端
- **渐进增强**：核心功能优先，高级功能逐步迭代

## 2. 技术选型

### 2.1 前端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| **Next.js** | 15+ | React 全栈框架，App Router |
| **React** | 19+ | UI 库 |
| **TypeScript** | 5+ | 类型安全 |
| **Ant Design** | 5+ | 企业级组件库 |
| **Tailwind CSS** | 3+ | 原子化样式 |
| **Zustand** | 4+ | 状态管理 |
| **TanStack Query** | 5+ | 服务端状态缓存 |
| **React Hook Form** | 7+ | 表单管理 |
| **Zod** | 3+ | 数据验证 |
| **Framer Motion** | 10+ | 动画过渡 |

### 2.2 后端技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| **FastAPI** | 0.100+ | HTTP API（扩展现有 FastMCP） |
| **SQLAlchemy** | 2+ | ORM |
| **Alembic** | 1+ | 数据库迁移 |
| **Pydantic** | 2+ | 数据验证 |
| **httpx** | 0.24+ | 异步 HTTP 客户端 |

### 2.3 数据存储

| 存储类型 | 技术 | 用途 |
|----------|------|------|
| **关系数据库** | SQLite | 会话、生成记录、工作流模板 |
| **文件系统** | 本地目录 | 图片、视频、音频素材 |
| **缓存** | TanStack Query | 前端 API 响应缓存 |

### 2.4 AI 接口适配

| 后端 | 生图 | 生视频 | 备注 |
|------|------|--------|------|
| OpenAI 兼容接口 | ✅ | ❌ | DALL-E 3 / Flux |
| ComfyUI | ✅ | ✅ | 本地部署，灵活可控 |
| 火山方舟 | ✅ | ✅ | Seedance 2.0 |
| Kling API | ❌ | ✅ | 视频生成（预留） |

# sessions 会话表
class Session:

# AI 生成
POST   /api/generate/image        # 文生图

# 图像处理

#### 3.2.4 AI 接口适配器

**接口定义**：

```python
class ImageGenerator(Protocol):
    async def text2img(
        self,
        prompt: str,
        negative_prompt: str,
        width: int,
        height: int,
        num_images: int,
        steps: int,
        cfg_scale: float,
        seed: int,
    ) -> GenerationResult: ...

    async def img2img(
        self,
        image: bytes,
        prompt: str,
        negative_prompt: str,
        denoising_strength: float,
        width: int,
        height: int,
        num_images: int,
        steps: int,
        cfg_scale: float,
        seed: int,
    ) -> GenerationResult: ...

class GenerationResult:
    images: list[bytes]
    seeds: list[int]
    info: dict
```

**实现**：
- `OpenAIGenerator`：对接 OpenAI 兼容接口

## 4. 实现路径

### 4.1 阶段划分

| 阶段 | 内容 | 预估周期 |
|------|------|----------|
| **Phase 1** | 基础架构：前后端项目初始化、数据库、API 骨架 | 1 周 |
| **Phase 2** | 生图功能：工作流、文生图、图生图、AI 适配器 | 2 周 |
| **Phase 3** | 视频生成：接口预留、UI 占位 | 2 天 |
| **Phase 4** | 图像工具：抠图、视频转序列帧、SpriteSheet 拆分 | 1 周 |
| **Phase 5** | 会话与素材管理：历史记录、素材库 | 1 周 |

### 4.2 迁移策略

1. **保留现有 v2f 功能**：将 `v2f_static.py` 的 HTML/JS 逻辑迁移到 React 组件
2. **后端 API 兼容**：新增 API 路由，不破坏现有 FastMCP 接口
3. **数据库渐进式引入**：先文件存储，后续迁移 SQLite

### 4.3 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| AI 接口不稳定 | 生成失败 | 重试机制、多后端切换 |
| 大文件存储 | 磁盘占用 | 定期清理、压缩存储 |
| 前端性能 | 大量图片渲染卡顿 | 虚拟列表、懒加载 |
